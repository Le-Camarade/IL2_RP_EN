# CLAUDE.md — IL-2 Career Narrator (PWCG)

## Context

Narrative roleplay module for IL-2 Sturmovik career mode, driven by **PWCG** (Pat Wilson's Campaign Generator). Claude Code transforms raw PWCG data + IL-2 logs into persistent narration: briefings, debriefings, squadron life.

Main interface: **terminal** via Claude Code.

---

## Player pilot

To be determined automatically via PWCG files when the campaign is launched. The `parse_pwcg.py` script identifies the player pilot, their aircraft, unit, and campaign.

---

## Configuration — .env

The `.env` file at the root contains the paths to the IL-2 and PWCG data. Scripts read these paths automatically.

```env
# IL-2 folder containing missionReport*.txt
IL2_DATA_DIR=C:\Program Files\IL-2 Sturmovik Great Battles\data

# Active PWCG campaign folder
PWCG_CAMPAIGN_DIR=C:\...\PWCGBoS\User\Campaigns\MaCampagne
```

IL-2 prerequisite: `mission_text_log = 1` in `startup.cfg` (section `[KEY = system]`).

## Data sources

### 1. PWCG files (JSON)

Typical path: `<IL-2 install>/PWCGBoS/User/Campaigns/<CampaignName>/`

| File | Content | Usage |
|---------|---------|-------|
| `Campaign.json` | Campaign date, map, mode | Temporal and geographical context |
| `Personnel/<SquadronID>.json` | Pilots: name, rank, skill, victories, status | Pilot sheets, dispersal |
| `CombatReports/<PilotSerial>/YYYYMMDD.CombatReport.json` | PWCG post-mission report | Cross-reference with debrief |
| `MissionData/Nicky Falstaff II YYYY-MM-DD.MissionData.json` | Generated mission parameters (duty, weather, aircraft, escort) | Briefing |
| `Equipment/` | Aircraft available per squadron | Equipment context |

### 2. .Mission file (structured text)

Generated by PWCG in `data/Multiplayer/Cooperative/`. Contains the full structure: waypoints, objectives, aircraft, weather. Text format with nested blocks (`Plane {}`, `MCU_Waypoint {}`, `Options {}`).

### 3. missionReport logs (text, AType)

Generated by IL-2 in `<IL-2 install>/data/` if `mission_text_log = 1` in `startup.cfg`. One mission produces **several numbered files**:

```
missionReport(2026-05-15_20-30-45)[0].txt
missionReport(2026-05-15_20-30-45)[1].txt
...
```

Each line follows the format: `T:<timestamp> AType:<N> <keys:values>`

Critical ATypes:

| AType | Event | Key fields |
|-------|-----------|-------------|
| 0 | MissionStart | GDate, GTime, MFile |
| 1 | Hit | AID (attacker), TID (target) |
| 2 | Damage | AID, TID, damage percentage |
| 3 | Kill | AID (attacker), TID (victim) |
| 5 | TakeOff | ID |
| 6 | Landing | ID |
| 11 | Group | GID (group), IDS (members) |
| 12 | ObjectSpawned | ID, TYPE, COUNTRY, NAME, POS |

**COUNTRY:102** = Allies | **COUNTRY:201** = Axis

> **Parsing note:** AType:11 entries define enemy aircraft groups. An enemy aircraft and its bot pilot share the same group, their IDs differing by 1024. To identify aircraft types, look for intermediate AType:12 entries (mid-mission spawns). Python scripts may miss kills if AType:11 groups are not correctly resolved — **always read the raw logs manually** before relying on script output.

## Workflows — IO prompts and orchestration

Three keyword-triggered workflows. Each workflow is documented in detail in `resources/`.

| Keyword | Workflow | Detailed prompt |
|---------|----------|-----------------|
| `briefing` | Pre-mission briefing | `resources/prompt-io-briefing.md` |
| `debrief` | Interactive post-mission debriefing | `resources/prompt-io-debrief.md` |
| `dispersal` | Dispersal hut roleplay | `resources/prompt-dispersal.md` |

Sequencing logic, error handling, log selection: `resources/prompt-orchestration.md`.

### Briefing — summary

1. Run `python scripts/post_mission.py fix-lang` (silent) — generates language files `.fra`, `.ger`, etc. for the PWCG mission
2. Read `Campaign.json` to get the campaign date (`YYYYMMDD`)
3. Read `MissionData/Nicky Falstaff II YYYY-MM-DD.MissionData.json` (current campaign date)
   - If absent: contextual briefing only + note to player to generate the mission in PWCG
3. Read `squadron/tableau-de-bord.md` + latest entries from `squadron/journal.md`
4. Save the briefing to `missions/briefings/YYYY-MM-DD_briefing-NN.md`
5. Display the briefing **in character as IO** (structure in `prompt-io-briefing.md`)
6. Answer the player's questions in character
7. When the pilot leaves to fly: "Understood. See you at the debrief."

**IO briefing rules:**
- All units in **feet and mph** (never metres or m/s) — convert PWCG data
- Never name enemy pilots or give their precise strength before combat
- Expected opposition = probable units and aircraft types (historical context), not enemy MissionData
- **During debrief too**: the IO does not know the names of enemy pilots killed during the interrogation or when announcing confirmed kills — identification takes several days (separate intelligence channels)

### Debriefing — summary

1. **Read the raw logs manually** (silent):
   - List all `missionReport(*)*.txt` in `IL2_DATA_DIR`, identify the most recent session by timestamp
   - Read `[0].txt` → extract aircraft IDs (AType:12/10, ISPL:1 for the player)
   - Search all files for: AType:3 (kills), AType:2 (allied damage), AType:6 (landings/crashes), AID:-1 (flak)
   - Intermediate AType:12 entries identify enemy aircraft (type + name)
2. Run `python scripts/parse_mission_report.py` (supplement — may miss kills on AType:11 groups)
3. Run `python scripts/parse_pwcg.py` (silent)
4. Store the data — **do not show it to the player**
4. Interrogation **in character as IO**, one question at a time (6 phases in `prompt-io-debrief.md`)
5. Cross-reference the player's account with the logs. Normal fog of war.
6. Announce confirmed results
7. Generate files:
   - `missions/YYYY-MM-DD_mission-NN.md` — RAF Combat Report
   - Update `personnel/` (pilot, allies hit, enemies killed)
   - `squadron/journal.md` — short entry
   - `squadron/tableau-de-bord.md` — updated stats
   - `squadron/memorial.md` — if fallen
8. Run `python scripts/post_mission.py journal YYYYMMDD "text"` — injects the `squadron/journal.md` entry into PWCG's `CampaignLog.json`

### Dispersal — summary

1. Read `squadron/tableau-de-bord.md`, `squadron/journal.md`, `personnel/allies/*.md`, latest `missions/*.md`
2. Choose 2-4 pilots present (consistent with context)
3. Set the scene in 2-3 sentences
4. Bring the pilots to life — each reacts according to their profile, not a script
5. The player speaks whenever they want, pilots also talk among themselves
6. At the end: entry in `squadron/journal.md` + update pilot sheets if any development

## Narrative tone

- **Combat Reports**: form inspired by RAF Form 540/F, written in English. Sober, factual, dated.
- **Dispersal journal**: Pierre Clostermann (*Le Grand Cirque*). Human, precise, dry wit.
- **Enemy pilot sheets**: fictional biographies consistent with the period and unit.
- **Allied pilots**: emerging personalities based on their history.
- **Language**: everything in English.

---

## Constraints

- Everything stored **locally**, no cloud
- Markdown files are readable without any special tool
- Do not invent data absent from the logs — flag gaps
- Tone remains sober and military, never heroic
- Enemy pilots are treated with respect — the memorial makes no distinction between sides
